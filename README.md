# RAG Paper Compare

RAG Paper Compare is a small end-to-end AI-enabled application for comparing a fixed set of RAG research papers. It is intentionally narrow: the app supports guided comparison questions over three curated papers, with cited evidence pulled from processed corpus files.

## What the App Does

- Ingests a fixed set of public paper PDFs from `data/bronze/paper-sources.json`
- Runs ETL to normalize extracted text into staged corpus artifacts
- Stores curated application data as repo-backed JSON in `data/gold/`
- Uses a server-side LLM call to generate structured paper comparisons
- Exposes a guided UI for comparing 2-3 papers with evidence snippets

## What It Does Not Support

- User-uploaded files
- Open-ended web research
- Arbitrary chat over unrelated topics
- Production-scale retrieval infrastructure

## Architecture

The data flow is deliberately simple:

1. `Bronze`: source metadata and PDF URLs
2. `Silver`: extracted paper text generated by the ingestion script
3. `Gold`: curated `papers.json` and `chunks.json` used by the app
4. `API`: `POST /api/compare` validates the request, selects evidence, and calls the server-side reasoning layer
5. `UI`: the Next.js page renders a guided comparison experience for supported prompts

## Local Setup

1. Install dependencies:

```bash
npm install
```

2. Add environment variables in `.env.local`.

OpenAI:

```bash
LLM_PROVIDER=openai
OPENAI_API_KEY=your_key_here
OPENAI_MODEL=gpt-4o-mini
```

Groq:

```bash
LLM_PROVIDER=groq
GROQ_API_KEY=your_key_here
GROQ_MODEL=llama-3.3-70b-versatile
```

3. Run the app:

```bash
npm run dev
```

4. Optional: regenerate the corpus:

```bash
npm run ingest
```

## Tests

- Unit and API behavior tests:

```bash
npm run test
```

- Browser-based Playwright test:

```bash
npm run test:e2e
```

- Production build:

```bash
npm run build
```

## GitHub Actions ETL

The repo includes `.github/workflows/ingest.yml`, which runs the ingestion pipeline and uploads the staged corpus artifacts as a workflow artifact.

## Deployment

Deploy the repo to Vercel and set the same environment variables there:

- OpenAI: `LLM_PROVIDER=openai`, `OPENAI_API_KEY`, `OPENAI_MODEL`
- or Groq: `LLM_PROVIDER=groq`, `GROQ_API_KEY`, `GROQ_MODEL`

The deployed app should expose the same guided comparison workflow shown locally.
